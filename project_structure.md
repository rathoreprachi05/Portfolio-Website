# Project Structure Overview

This document provides a organized high-level mapping of your Next.js portfolio project to help you easily understand where everything is located and what each file/folder is used for!

## 📂 Root Directory
The root manages your application configurations, dependencies, and environments.

```text
📁 Website Portfolio
├── 📄 package.json          - Lists all your installed dependencies/libraries (like Next.js, React, Framer Motion) and custom scripts (dev, build, lint).
├── 📄 tsconfig.json         - Configuration for TypeScript compilation and paths.
├── 📄 next.config.ts        - Core settings for Next.js framework (like image domains, redirects, custom webpack setup).
├── 📄 tailwind / postcss    - (e.g. `postcss.config.mjs`) Configures Tailwind CSS V4 / PostCSS processing.
├── 📄 eslint.config.mjs     - Linter configuration to keep your code standard and bug-free.
└── 📄 README.md             - Documentation covering project setup and notes.
```

## 📂 `src/` (Source Code)
This is where all your actual application logic and UI lives.

```text
📁 src
├── 📁 app                   - App Router features (Routing, Layouts, global styles).
│   ├── 📄 page.tsx          - The main landing page / index view joining your sections.
│   ├── 📄 layout.tsx        - The HTML "shell" wrapping every page (handles fonts, metadata, providers).
│   └── 📄 globals.css       - Your root stylesheet containing Tailwind imports and CSS variables (theme colors).
│
├── 📁 components            - Your reusable React UI pieces.
│   ├── 📁 home              - Sections explicitly related to your homepage (e.g., `Hero.jsx`, `AboutBento.jsx`, and soon `Showcase.jsx`).
│   ├── 📁 ui                - Generic, granular design system parts (e.g., `Button.tsx`, inputs, cards).
│   └── 📁 layout            - Wrappers for Navigation bars, footers, etc.
│
└── 📁 lib                   - Utility scripts, helper functions, and shared constants.
```

## 📂 `public/` (Public Assets)
Files here are publicly accessible via the browser at the base URL (e.g., `yourwebsite.com/assets/...`).

```text
📁 public
└── 📁 assets                - SVG layouts, logos, background shapes, or icons (`globe.svg`, `file.svg`, etc). 
                               (Note: The Showcase mockups will be generated and placed here!).
```

## 📂 `Assets/`, `Wireframes/`, `Documentations/` (Raw Assets)
These exist outside the source code as a private staging area. They are naturally used to organize your assets before deciding to ship them inside `public/`.

```text
📁 Assets                    - Contains unprocessed or raw images (`Animated Me gamer.png`, cover banners, etc).
📁 Wireframes                - Usually holds Figma exports or layout designs.
📁 Documentations            - Standard planning logic and conceptual markdown files.
```

## 📂 `.next/` and `node_modules/` (Autogenerated)
You never manually edit these folders!
- **`.next/`**: Temporarily stores compiled and cached build files generated by Next.js when running `npm run dev` or `build`.
- **`node_modules/`**: Contains the source code for all NPM packages defined in your `package.json`. It is huge and ignored by git!

---

> [!TIP]
> Whenever you want to create a **new Section**, it goes in `src/components/home/`.
> Whenever you introduce **new images** to be displayed on the page via `<Image>`, put them in `public/assets/`.
> Whenever you want to tweak **global CSS themes**, edit `src/app/globals.css`.
